### New features

- Typed the `build_processing` queue-job `progress` payload. A new `BuildProcessingProgress` model — with nested `EditionUpdateRef` (the `editions_updated`/`editions_skipped` entries) and `PublishJobRef` (the `publish_jobs` entries) — is exported from `docverse.client.models`, and `QueueJob.progress` is now `BuildProcessingProgress | None` instead of an untyped dict, so generated clients and `api-types` get real fields. All fields are optional and the models use `extra="allow"`, so other job kinds' progress payloads round-trip unchanged.
