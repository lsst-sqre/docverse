### New features

- New `LifecycleRule` discriminated union (`DraftInactivityRule`, `BuildHistoryOrphanRule`, `RefDeletedRule`) and `LifecycleRuleSet` root model exported from `docverse.client.models`. The `lifecycle_rules` field on `Organization`, `OrganizationCreate`, `OrganizationUpdate`, `Project`, `ProjectCreate`, and `ProjectUpdate` now uses this typed shape so script-side configuration gets IDE autocomplete and pydantic validation, and the duplicate-rule-type guard runs locally before any request goes out. `ProjectCreate` gains the field so a project can be provisioned with lifecycle rules in a single request instead of requiring a follow-up PATCH.
