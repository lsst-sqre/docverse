### New features

- The `lifecycle_eval` evaluator's `ref_deleted` branch is no longer a stub: a draft edition tracking a literal git ref (`tracking_mode` of `git_ref` or `alternate_git_ref`) is now matched for soft-delete when its `tracking_params['git_ref']` is absent from the project's live ref set. The evaluator accepts the live ref set through a new `LifecycleEvaluationContext.live_refs: frozenset[str] | None` field; `None` means "no ref information available" and produces no matches, while an empty `frozenset()` means "the repo currently has no branches or tags" and matches every eligible draft. `evaluate_lifecycle()` now takes a single `LifecycleEvaluationContext` parameter bundling editions, builds, edition-build-history rows, `now`, and `live_refs`. The existing `lifecycle_eval` worker passes `live_refs=None`; the GitHub webhook fast-path and `git_ref_audit` cron added in later DM-54913 slices populate the field.
