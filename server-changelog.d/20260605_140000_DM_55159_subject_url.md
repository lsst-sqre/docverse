### New features

- `GET /queue/jobs/{id}` (and the keeper-sync run-jobs list) now return `subject_url`, `build_url`, and `edition_url` back-references on each queue job: absolute HATEOAS links to the API resource the job processes — the build for `build_processing` jobs, the edition for publish jobs — resolved at request time from the job's stored identifiers via the factory stores. Each field is `null` when the job targets no such resource or it could not be resolved (e.g. a deleted build/edition).
