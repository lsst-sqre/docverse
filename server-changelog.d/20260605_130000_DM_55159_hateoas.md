### New features

- The `build_processing` queue-job `progress` payload now embeds self-describing HATEOAS links: each `editions_updated[]` entry carries an absolute `edition_url` and each `publish_jobs[]` entry an absolute `queue_job_url`, so clients follow links instead of reconstructing `…/editions/{slug}` and `/queue/jobs/{id}` paths. The worker resolves the Docverse API base URL from Rubin Repertoire (`DiscoveryClient.url_for_internal("docverse")`). When Docverse is not registered in Repertoire the build still completes — the URL fields are omitted and a warning is logged. A new `docverse.domain.api_urls` module composes the resource URLs from a base plus identifiers, with a drift-guard test pinning its templates to the live `get_edition` / `get_build` / `get_queue_job` routes.
