### Backwards-incompatible changes

- `PATCH /orgs/{org}` and `PATCH /orgs/{org}/projects/{project}` now validate `lifecycle_rules` against a typed discriminated-union schema. `POST /orgs/{org}/projects` accepts the same typed `lifecycle_rules` field so a project can be provisioned with lifecycle rules without a follow-up PATCH. Unknown rule types, missing required fields, or duplicate rule kinds within one set now return 422 at the API boundary instead of being silently persisted as opaque JSONB. The `Organization`, `Project`, and corresponding `*Update`/`*Create` payloads return `lifecycle_rules` under the same typed shape.
