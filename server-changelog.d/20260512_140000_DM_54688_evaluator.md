### New features

- Added the pure `LifecycleEvaluator` for the upcoming `lifecycle_eval` background job (SQR-112). Given a resolved `LifecycleRuleSet` plus a project's editions, builds, and per-edition build history, `evaluate_lifecycle()` returns the set of edition ids and build ids to soft-delete and per-rule match counts for structured logging. The function performs no I/O so DM-54914's LTD-sync path can reuse it directly with pre-fetched data. A `resolve_rule_set()` helper applies the SQR-112 inheritance rule that a project rule list, when set, entirely replaces the organization-level rules with no merging.
